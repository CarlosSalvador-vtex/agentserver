## Bloqueador identificado

Ingress wildcard `*.agentserver.analytics.vtex.com` aponta para `agentserver-sandboxproxy`. sandboxproxy (`internal/sandboxproxy/server.go:108-119`) só matchea prefixos `claw-` e `hermes-`. Demais subdomínios caem em chi 404.

Consequência: `<slug>.agentserver.analytics.vtex.com/login` retornava **404**.

### Fix implementado (PR #58)

`internal/sandboxproxy/`:

```go
// config.go — novo env
AgentserverUpstream string  // AGENTSERVER_UPSTREAM
```

```go
// server.go — Server gains a fallback reverse proxy
type Server struct {
    // ...
    AgentserverFallback *httputil.ReverseProxy
}

func New(cfg Config, ...) *Server {
    // ...
    if cfg.AgentserverUpstream != "" {
        u, _ := url.Parse(cfg.AgentserverUpstream)
        rp := httputil.NewSingleHostReverseProxy(u)
        origDirector := rp.Director
        rp.Director = func(req *http.Request) {
            origHost := req.Host
            origDirector(req)
            req.Host = origHost  // preserve Host for slug extraction
        }
        s.AgentserverFallback = rp
    }
    // ...
}
```

No middleware de roteamento, depois dos matches `claw-*` / `hermes-*`:

```go
if sub != "" && s.AgentserverFallback != nil {
    s.AgentserverFallback.ServeHTTP(w, r)
    return
}
```

Helm template `sandboxproxy.yaml` injeta:

```yaml
- name: AGENTSERVER_UPSTREAM
  value: "http://{{ .Release.Name }}:{{ .Values.service.port }}"
```
