## Architecture: Data Flow

```
Sister's Browser
  │
  ├─ Step 1: /playground (re-enabled)
  │    TopBar.tsx → "Configurar Agente" nav link
  │    App.tsx → /playground, /playground/souls/:id, /playground/skills/:id routes
  │
  ├─ Step 2: "Usar modelo de cobrança"
  │    Playground.tsx → shows marketplace button
  │    GET /api/marketplace/souls  (visibility='shared', ws NULL)
  │    POST /api/marketplace/souls/{id}/fork  {workspace_id: sis-ws}
  │      → ForkSoulDraft(sourceID, sister.userID, sister.wsID)
  │      → soul_drafts INSERT (author=sister, ws=sister-ws, name="Agente de Cobrança-fork")
  │    UI strips "-fork" suffix, sister renames → PATCH /api/playground/souls/{id}
  │
  ├─ Step 3: "Personalidade do agente" editor
  │    /playground/souls/:id → PlaygroundSoulEditor (isDevMode=false)
  │    Visible: Name (editable), Tone/formality dropdown, Instructions body textarea
  │    Hidden: promote button, visibility toggle, dry-run panel, PR diff badge,
  │            voice.language, voice.tone_examples, constraints.*, compatible_skills
  │    Save → PATCH /api/playground/souls/{id}  {frontmatter, body}
  │
  ├─ Step 4: "Publicar agente"
  │    deployAgent(wsId, soulId, skillIds) in lib/deploy.ts:
  │      1. listSandboxes(wsId)
  │      2. for each existing sandbox: DELETE /api/sandboxes/{id}
  │      3. createSandbox(wsId, "Agente Produção", "openclaw", ...,
  │           composition: {soul: "draft:<soulId>", skills: ["draft:<skillId>"]})
  │      error paths:
  │        - quota_exceeded → "Workspace sem cota — peça ao administrador"
  │        - delete 404 → treat as success (sandbox already gone)
  │        - list failure → surface error, do NOT proceed to delete
  │
  └─ Step 5: WhatsApp binding (ops)
       POST /api/workspaces/{id}/im/whatsapp/configure {phone_number_id, access_token}
       POST /api/sandboxes/{id}/im/bind {channel_id}
```

### Step 4: Deploy Button (~1.5 days with D1/D5 included)

**web/src/lib/deploy.ts** (NEW FILE)
```typescript
export async function deployAgent(
  workspaceId: string,
  soulId: string,
  skillIds: string[],
): Promise<{ sandboxId: string }> {
  // 1. List and delete existing sandboxes
  const sandboxes = await listSandboxes(workspaceId)
  await Promise.all(sandboxes.map(s =>
    deleteSandbox(s.id).catch(e => {
      if (e instanceof ApiError && e.status === 404) return  // already gone
      throw e
    })
  ))
  // 2. Create new with composition
  try {
    const sandbox = await createSandbox(workspaceId, 'Agente Produção', 'openclaw',
      undefined, undefined, undefined, undefined,
      { soul: `draft:${soulId}`, skills: skillIds.map(id => `draft:${id}`) }
    )
    return { sandboxId: sandbox.id }
  } catch (e) {
    if (e && typeof e === 'object' && 'error' in e && e.error === 'quota_exceeded') {
      throw new Error('Workspace sem cota para sandboxes — peça ao administrador para configurar.')
    }
    throw e
  }
}
```

## Edge Cases Handled

| Case | Handling |
|------|----------|
| Sister clicks "Publicar" twice | Second call: delete succeeds (sandbox may still be creating), create new — idempotent by design |
| Quota not set by admin | `deployAgent` catches `quota_exceeded`, shows PT-BR message |
| Admin forks template for sister | Sister gets 403 on PATCH. **Constraint: sister must fork herself.** |
| WhatsApp 24h window | Meta policy: if debtor doesn't message in 24h, agent can't initiate. Documented as known limitation. |
| Template not found in marketplace | `listMarketplaceSouls` returns empty → show "Nenhum modelo disponível" |
| Sandbox delete 404 during deploy | Treated as success — sandbox was already gone |
